# Contributing to SecDeploy-Audit

Thank you for your interest in improving SecDeploy-Audit! We welcome contributions from everyone. Here is how you can get started.

## How to Contribute

### 1. Reporting Bugs & Requesting Features
* Check the **Issues** tab to see if your topic is already being discussed.
* If not, open a new issue. Describe the problem clearly or detail the feature request.

### 2. Submitting Code Changes
1. **Fork** this repository.
2. **Clone** your fork locally: `git clone'
3. Create a new branch for your work: `git checkout -b feature/your-feature-name`
4. Make your changes. Ensure your code is clean and follows project styles.
5. Push your branch to your fork: `git push origin feature/your-feature-name`
6. Open a **Pull Request (PR)** against our `main` branch.

## Code Review Process
Once you submit a PR, we will review it as soon as possible. Please ensure:
* Your PR description clearly explains what changes were made.
* Any related open issue is linked in the description.
